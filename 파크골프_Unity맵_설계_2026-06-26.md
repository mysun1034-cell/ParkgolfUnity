## 0. 한 줄 결론

문제는 "맵 한 개가 어렵다"가 아니라 **"만들 맵 양이 수백 개"** 그리고 "한국 외주가 비싸다"이다.
정답은 **외주처를 싸게 찾는 게 아니라 "코스 = 데이터(스플라인 경로 + 폭 + 설정값)"로 정의하고
한 번 만든 절차생성 파이프라인에 데이터만 입력해 코스를 양산**하는 구조로 바꾸는 것.
에셋은 거의 전부 **무료 CC0**로 충당 가능하고, 외주가 정말 필요하면 **모듈 키트 1회만** 발주한다.

---

## 1. 전략 — 외주 풀제작은 처음부터 막힌다

- 스튜디오 기준 저폴리 환경 1식 = **$1,500~5,500**. 코스마다 풀제작 × 수백 개 = 비현실적.
- 따라서 비용 구조를 바꿔야 한다:
  1. **무료 CC0 에셋**으로 부품 확보(비용 0).
  2. **절차생성 파이프라인 1회 구축**(여기에 투자).
  3. 코스 = 스플라인 + 좌표 **데이터 입력**만 → 코스당 한계비용 ≈ 0.
  4. 외주가 필요하면 **모듈 키트만 1회** 발주($수천), 배치는 직접/저가로.

## 3. 절차생성 파이프라인 설계

### 3.0 데이터 흐름

```
 작성(둘 중 하나)                     생성 파이프라인(1회 제작, 모든 코스 재사용)         결과
┌─────────────────┐
│ A. 디자이너 손작성 │   ScriptableObject ┐
│  (씬에서 스플라인) │ ─────────────────→ │
└─────────────────┘                     ├─→ CourseBuilder ─┬─→ ① 지형(Terrain): DEM RAW 임포트 or 평지
┌─────────────────┐                     │                 ├─→ ② 페어웨이/그린/티: 스플라인→리본 메시
│ B. 실측 자동수입  │   JSON DTO ────────┘                 ├─→ ③ 컵/깃대: cupPosition에 프리팹 배치
│ (위성 트레이싱·DEM)│                                       └─→ ④ 나무·벤치·울타리: 규칙 기반 산포
└─────────────────┘                                              (Kenney/Quaternius CC0 에셋)
```

### 3.1 데이터 스키마

손작성 = ScriptableObject(강타입·인스펙터), 실측 대량수입 = JSON DTO.
검증: `JsonUtility`는 최상위 배열 불가 → 반드시 래퍼 객체. 복잡 스키마는 `com.unity.nuget.newtonsoft-json`(Json.NET 13.0.1).

```csharp
using UnityEngine;
using UnityEngine.Splines;

public enum HoleHazard { None, Sand, Water }

// ── 디자이너 손작성용 (Assets ▸ Create ▸ ParkGolf ▸ Hole Definition) ──
[CreateAssetMenu(fileName = "Hole", menuName = "ParkGolf/Hole Definition")]
public class HoleDefinition : ScriptableObject
{
    [Header("규준 파라미터")]
    [Range(1, 9)] public int holeNumber = 1;
    public int   par = 3;                       // 파크골프 9홀: 파3×4 + 파4×4 + 파5×1 = 33타
    [Tooltip("규준 상한 — 파3 ≤60 / 파4 ≤100 / 파5 ≤150 (m). 하한은 시설 재량(휴리스틱)")]
    public float distanceMeters = 50f;
    [Tooltip("페어웨이 폭(m). 규준 최소 2m")]
    public float fairwayWidth = 4f;
    public HoleHazard hazard = HoleHazard.None;

    [Header("형상 데이터")]
    public Spline centerline;                   // Spline은 [Serializable] → SO에 직접 저장 가능
    public Vector3 teePosition;                 // (SplineContainer는 MonoBehaviour라 SO 저장 불가)
    public Vector3 cupPosition;
}

[CreateAssetMenu(fileName = "Course", menuName = "ParkGolf/Course Definition")]
public class CourseDefinition : ScriptableObject
{
    public string courseName;
    public HoleDefinition[] holes;              // 보통 9개
    [Header("실측 지형(선택)")]
    public string heightmapRawPath;             // DEM에서 만든 .raw (3.3절)
    public Vector3 terrainSize = new Vector3(200, 30, 200); // 폭/최대표고/길이(m)
}
```

```csharp

[System.Serializable] public class Vec3   { public float x, y, z; }
[System.Serializable] public class HoleDTO {
    public int holeNumber, par;
```

> 작성 동선: 홀별 스플라인은 씬의 `SplineContainer` 프리팹에서 편집(Unity 스플라인 툴이 거기서만 동작) →
> "Bake" 버튼으로 `HoleDefinition.centerline`(Spline)에 구워 넣기. JSON 경로는 `centerlinePoints` → `BezierKnot` 추가로 스플라인 생성.
> (`BezierKnot`은 `[Serializable]` struct: Position/TangentIn/TangentOut = float3. 자동스무딩 탄젠트 API는 채택 시 버전 문서로 재확인.)

### 3.2 페어웨이 메시 생성기 — 스플라인 → 평평한 리본

기본 `SplineExtrude`는 **튜브(원형 단면)만** 됨(확인됨) → 평평한 페어웨이는 커스텀 메시 필요.
아래는 전부 **공식 문서로 confirmed된 시그니처**만 사용.

```csharp
using System.Collections.Generic;
using Unity.Mathematics;
using UnityEngine;
using UnityEngine.Splines;

[RequireComponent(typeof(MeshFilter), typeof(MeshRenderer))]
public class RibbonMeshBuilder : MonoBehaviour
{
    public SplineContainer spline;      // 홀 중심선
    public float width   = 4f;          // 폭(m) — 규준 최소 2m
    public float spacing = 1f;          // 길이방향 샘플 간격(m)

    public void Build()
    {
        var s = spline.Spline;
        // CalculateLength<T>(this T spline, float4x4 transform) → float  [confirmed]
        float length = SplineUtility.CalculateLength(s, spline.transform.localToWorldMatrix);
        int segments = Mathf.Max(2, Mathf.RoundToInt(length / spacing));

        var verts = new List<Vector3>((segments + 1) * 2);
        var uvs   = new List<Vector2>((segments + 1) * 2);
        var tris  = new List<int>(segments * 6);

        for (int i = 0; i <= segments; i++)
        {
            float t = (float)i / segments;
            // SplineContainer.Evaluate(float t, out float3 pos, out float3 tan, out float3 up) → bool
            // (컨테이너 멤버 = 월드 좌표 반영)  [confirmed]
            spline.Evaluate(t, out float3 pos, out float3 tan, out float3 up);

            // right = normalize(cross(normalize(tan), normalize(up)))   [medium — 부호 주의]
            float3 right = math.normalize(math.cross(math.normalize(tan), math.normalize(up)));
            float3 l = pos - right * (width * 0.5f);
            float3 r = pos + right * (width * 0.5f);

            verts.Add(transform.InverseTransformPoint((Vector3)l));
            verts.Add(transform.InverseTransformPoint((Vector3)r));
            float v = length * t;
            uvs.Add(new Vector2(0, v)); uvs.Add(new Vector2(1, v));
        }

        for (int i = 0; i < segments; i++)
        {
            int b = i * 2;                               // 인접 좌우 4정점
            tris.Add(b); tris.Add(b + 2); tris.Add(b + 1);
            tris.Add(b + 1); tris.Add(b + 2); tris.Add(b + 3);
        }

        var mesh = new Mesh { name = "Fairway" };
        if (verts.Count > 65535)                          // 정점 한도 초과 시
            mesh.indexFormat = UnityEngine.Rendering.IndexFormat.UInt32;
        mesh.SetVertices(verts);
        mesh.SetTriangles(tris, 0);
        mesh.SetUVs(0, uvs);
        mesh.RecalculateNormals();
        mesh.RecalculateBounds();
        GetComponent<MeshFilter>().sharedMesh = mesh;
    }
}
```

> ⚠️ `right` 벡터의 `cross` 부호는 좌표계·winding에 따라 뒤집힘(medium). 메시가 아래를 보거나 안 보이면
> 삼각형 winding을 뒤집거나 `right` 부호 반전(`-right`). 같은 생성기를 페어웨이/그린/티/카트길에 폭·텍스처만 바꿔 재사용.

**대안(지형에 직접 칠하기):** DEM 지형 위에 스플랫맵으로 페어웨이 페인팅 →
`TerrainData.SetAlphamaps(int x, int y, float[,,] map)` (배열 `[y,x,레이어인덱스]`, 가중치 합=1),
`terrainData.terrainLayers`(TerrainLayer[]) 먼저 설정. (전부 confirmed)

### 3.3 DEM 실측 지형 배치 파이프라인

국토지리정보원 DEM 5m(IMG) → Unity Terrain 하이트맵. 검증된 GDAL 명령:

```bash
# 1) 재투영 — 한국 DEM을 미터단위 좌표계로 (EPSG:5186 = Korea 2000 중부원점)
gdalwarp -t_srs EPSG:5186 -r bilinear input_dem.img dem_5186.tif

# 2) 코스 영역 잘라내기 (-te xmin ymin xmax ymax, 단위=미터)
gdalwarp -te 200000 300000 200200 300200 dem_5186.tif dem_crop.tif

# 3) 16비트 RAW 생성 — Unity Terrain이 읽는 형식
gdal_translate -ot UInt16 -scale -of ENVI -outsize 1025 1025 dem_crop.tif heightmap.raw
#   -ot UInt16  : 16비트(0~65535)
#   -scale      : 입력 표고 min/max → 0~65535 정규화
#                 ★ 코스 간 절대표고 비교 필요하면 -scale DEMmin DEMmax 0 65535 로 고정
#   -outsize    : 반드시 2^n+1 (513 / 1025 / 2049 ...)  ← 1024 주면 Unity가 리샘플

# 4) 실표고 범위 확인 → Unity 매핑값
gdalinfo -mm dem_crop.tif        # Band min/max(미터) 확인
#   terrainData.size = new Vector3(폭_m, (DEMmax - DEMmin), 길이_m);
#   RAW의 0~65535(=정규화 0~1)가 0~size.y(미터)로 선형 매핑

# 5) 배치 — 코스 좌표 리스트(courses.csv) 반복
while IFS=, read xmin ymin xmax ymax name; do
  gdalwarp -te $xmin $ymin $xmax $ymax dem_5186.tif tmp.tif
  gdal_translate -ot UInt16 -scale -of ENVI -outsize 1025 1025 tmp.tif ${name}.raw
done < courses.csv
```

**검증된 2대 함정(실측 필수):**
1. **바이트오더** — ENVI 출력이 Windows에서도 big-endian으로 써지는 사례 → Unity RAW Import에서 **Byte Order "Mac"** 또는 swap.
2. **상하반전/회전** — GDAL 원점 top-left vs Unity Terrain bottom-left → 하이트맵 뒤집힘 → `numpy.flipud` 또는 임포트 후 보정.

> ⚠️ 정밀도 한계: 5m 격자라 홀(20~100m)당 표고점 4~20개 → **큰 기복만 OK**, 페어웨이 미세 굴곡 못 살림.
> → 지형 베이스 = DEM, 플레이 표면 = 3.2 메시의 하이브리드가 정답.
> 한국 좌표계: EPSG:5186(중부원점, NGII 표준) / 5179(UTM-K, 네이버) / 5174(구). 원본 SRS는 `gdalinfo`로 확인.

## 4. 파크골프 공식 규준 (생성 파라미터 · 출처/신뢰도)

> ★ 검증 결과: "9홀=33타"는 **파크골프 전용** 수치(일반 골프는 36). 우리는 파크골프이므로 33타가 맞음.

| 파라미터 | 값 | 신뢰도 | 비고 |
|---|---|---|---|
| 9홀 구성 | 파3×4 + 파4×4 + 파5×1 = **33타** | high | 18홀 66 / 27홀 99 / 36홀 132 (9홀 배수) |
| 홀 거리 **상한** | 파3 **≤60m** / 파4 **≤100m** / 파5 **≤150m** | high | 생성 max로 고정 |
| 홀 거리 하한 | 파3 ~40 / 파4 ~60 / 파5 ~100 | medium | 출처 편차 → 휴리스틱(시설 재량) |
| 9홀 총거리 | **≤790m** (=60×4+100×4+150) | medium | 산술 정합 |
| 페어웨이 폭 | 최소 **2m** | medium | 스플라인 width 하한 |
| 티 박스 | 1.5×1.5 ~ 2.0×2.0m | medium | |
| 그린 지름 | **≥5m** (원형) | medium | |
| 홀컵 | 내경 **200mm** / 외경 210mm | high | |
| 공 | 지름 **60mm**±0.5, 무게 ~90g | high(지름)/medium(무게) | 무게 80~100g 출처 편차 |
| 클럽 | 길이 **≤86cm**, 무게 ≤600g, 1인 1클럽 | high(길이)/medium(무게) | 550g 출처도 존재 |
| 코스 면적 | 9홀 ~7,500–8,250㎡ | low | 출처 상충 → 휴리스틱 |
| OB 말뚝 | 길이 40cm, 직경 4cm | low | 단일 출처 |

> 물리/렌더에 영향 주는 무게·면적은 **KPGA/KPGF 공인규정 원문(PDF)**으로 단일값 확정 권장

## 6. 검증된 Unity API 레퍼런스 (정확 시그니처)

> 모두 공식 docs.unity3d.com 교차검증 `confirmed`. Splines 2.0~2.8 동일.

**Unity Splines** (`UnityEngine.Splines`, `com.unity.splines`):
- `bool SplineContainer.Evaluate(float t, out float3 pos, out float3 tan, out float3 up)` — **월드 좌표**, t=0~1. (오버로드: `(int splineIndex, ...)`)
- `bool SplineUtility.Evaluate<T>(this T spline, float t, out float3 pos, out float3 tan, out float3 up) where T:ISpline` — **로컬 좌표**.
- `float3 SplineUtility.EvaluatePosition<T>(this T spline, float t)`
- `float3 SplineUtility.GetPointAtLinearDistance<T>(this T spline, float fromT, float relativeDistance, out float resultPointT)` — 등간격 샘플링.
- `float SplineUtility.CalculateLength<T>(this T spline, float4x4 transform)`
- `BezierKnot` = `[Serializable]` struct(Position/TangentIn/TangentOut = float3). `Spline`은 직렬화 가능, `SplineContainer`는 MonoBehaviour(SO 저장 불가).
- `SplineExtrude` = 원형 단면(tube/wire/rope)만, 평면 리본 불가.

**Unity Terrain** (`UnityEngine`):
- `void TerrainData.SetHeights(int xBase, int yBase, float[,] heights)` — 값 0~1, 인덱스 **[y,x]**, 실미터 = 값 × `size.y`.
- `int TerrainData.heightmapResolution` — **2^n+1**로 clamp(33/65/129/257/513/1025/2049/4097).
- `Vector3 TerrainData.size` — (폭, 최대표고, 길이) 월드 미터.
- `void TerrainData.SetAlphamaps(int x, int y, float[,,] map)` — **[y,x,레이어]**, 가중치 합=1.
- `TerrainLayer[] TerrainData.terrainLayers` — 3번째 차원 크기와 대응.
- `static GameObject Terrain.CreateTerrainGameObject(TerrainData)` — TerrainCollider 포함 생성.
- 스플라인→heightmap 인덱스 변환: `u=(P.x-pos.x)/size.x`, `ix=round(u*(res-1))` (휴리스틱, 공식 API 아님).

**Mesh** (`UnityEngine`):
- `Mesh.SetVertices` / `SetTriangles(list, submesh)` / `SetUVs(0, list)` / `RecalculateNormals()` / `RecalculateBounds()`.
- 정점 65535 초과 → `mesh.indexFormat = UnityEngine.Rendering.IndexFormat.UInt32`.

**기타:**
- JSON: `JsonUtility` 최상위 배열 불가(래퍼 필수) / 복잡 스키마 = `com.unity.nuget.newtonsoft-json` 13.0.1.
- 인스턴싱: `Graphics.RenderMeshInstanced` (구 `DrawMeshInstanced` 폐기, 호출당 511개).
- 스트리밍: `com.unity.addressables` `LoadAssetAsync`.
